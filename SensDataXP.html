Sensitive Data Exposure

3RD MOST CRITICAL SECURITY RISK*

Sensitive Data Exposure is a security risk where an API does not correctly protect sensitive data such as financial information, passwords, and other forms of personal information. The most frequent mistake that causes Sensitive Data Exposure is failing to encrypt sensitive data, and by acquiring these kinds of information,  attackers have the ability to commit crimes such as credit card fraud, identity theft, and other related crimes.  It is also important to note that recently attacks due to Sensitive Data Exposure have been very prevalent, so it is an important web security risk to be aware of.

The website sent an HTTP request to a server over an insecure connection. Your information was revealed in clear text and is vulnerable to snooping.  

Make sure that you always check your network traffic when using suspicious websites. You may notice strange HTTP requests to websites containing your information. If you are browsing using HTTP instead of HTTPS, then all of that information could be intercepted using a man-in-the-middle attack.

Ways to prevent this attack:

  • First and foremost, encrypt all sensitive data that is at rest!
  • Do not store sensitive data unless that is necessary, so get rid of sensitive data once it is no longer being used
  • Use secure algorithms and protocols, and also make sure you use appropriate key management
  • Do not allow caching for response that contains sensitive data because caching is another way that attackers can get access to sensitive data
  • Use salted hashing functions when storing passwords and other sensitive data
